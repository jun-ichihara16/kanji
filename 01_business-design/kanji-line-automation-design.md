# KANJI LINE自動化 & 振り返りヒアリング機能 設計ドキュメント

**作成日：** 2026年4月2日
**対象フェーズ：** フェーズ1後半〜フェーズ2

---

## 1. 機能の全体像

```
【幹事】
  ↓ イベント作成
  ↓ 参加者URLを送る（今まで通り）
  ↓ 参加者が登録 → KANJI公式を友達追加 → 幹事の「連絡先リスト」に自動蓄積

【次のイベント作成時】
  ↓ 過去の参加者リストが表示される
  ↓ 誰に送るかチェック
  ↓ 「幹事の名前っぽい文章」でLINEが自動送信される

【イベント当日〜翌日】
  ↓ KANJIが幹事にLINEで振り返りヒアリング
  ↓ 幹事が返答 → イベントメモとして保存

【参加者からの質問】
  ↓ KANJI公式LINEに質問が来る
  ↓ AIが該当イベントの情報を参照して自動返答
```

---

## 2. 機能詳細

---

### 機能A：参加者リストの自動蓄積

**トリガー：** 参加者がURLを踏んで登録完了した時

**フロー：**
1. 参加者が登録フォームに名前・支払い方法を入力
2. 登録完了画面で「LINEで支払い状況を受け取る」ボタンを表示
3. ボタンをタップ → KANJI公式アカウントの友達追加画面へ遷移
4. 友達追加完了 → Webhookでフォロー通知が届く
5. 参加者のLINE User IDと名前・電話番号を紐付けてDBに保存
6. 幹事の「連絡先リスト」に自動追加

**DB設計（追加テーブル）：**
```
contacts（幹事の連絡先リスト）
  - id
  - organizer_id（幹事のuser_id）
  - name（参加者の名前）
  - line_user_id（友達追加済みの場合）
  - phone_number（PayPay番号として登録済みの場合）
  - is_line_friend（友達追加済みフラグ）
  - first_event_id（初めて参加したイベントID）
  - last_event_id（最後に参加したイベントID）
  - event_count（参加イベント数）
  - memo（幹事が自由に書けるメモ）
  - created_at
  - updated_at
```

**友達追加状況の可視化：**
- ダッシュボードの参加者一覧に「LINE友達」バッジを表示
- 友達追加済み：緑バッジ
- 未追加：グレーバッジ（「LINEで通知を促す」ボタンを表示）

---

### 機能B：招待メッセージの自動送信

**トリガー：** 幹事がイベント作成後に「招待を送る」を実行した時

**フロー：**
1. 幹事がイベントを作成
2. 「招待を送る」ボタンをタップ
3. 過去の参加者リストが表示される（友達追加済みの人のみLINE送信可能）
4. 送りたい人をチェック
5. メッセージプレビューが表示される（AIが生成した自然な文章）
6. 幹事が確認・編集して「送信」
7. チェックした全員にLINEが一斉送信される

**メッセージの生成ルール：**
- 幹事の名前を文章内に必ず入れる（「市原さんより」ではなく「市原です」のトーン）
- イベント情報（店名・日時・参加費）を自然な文章に変換
- 毎回微妙に文章が変わるようにAIで生成（テンプレ感をなくす）
- 絵文字を適度に使用（bot感を減らす）

**メッセージ例（AI生成）：**
```
市原です！今度みんなで飲みに行こうと思って🍺
○月○日（土）19時〜、渋谷の「○○」を予約しました。
参加費は4,000円くらいの予定です。

参加できそうな人は下のリンクから登録してもらえると助かります👇
https://kanji.app/event/xxxxx

来れない場合も教えてもらえると嬉しいです！
```

**送信できない人への対応：**
- LINE未友達の参加者 → URLをコピーして手動送信するよう幹事に案内
- 「LINEで送れない人」として別リストに表示

---

### 機能C：参加者からの質問への自動返答

**トリガー：** 参加者がKANJI公式LINEにメッセージを送った時

**対応できる質問の例：**
- 「集合場所どこですか？」→ イベントの店名・住所を返す
- 「何時集合ですか？」→ 開始時刻を返す
- 「参加費いくらですか？」→ 参加費を返す
- 「まだ払ってないんですが大丈夫ですか？」→ 支払い方法と幹事の連絡先を返す
- 「行けなくなりました」→ 幹事に通知 + 「幹事に伝えました」と返す

**技術実装：**
- LINE Messaging API Webhookでメッセージを受信
- 参加者のLINE User IDから「どのイベントの参加者か」を特定
- イベント情報をコンテキストとしてLLMに渡す
- LLMが自然な返答文を生成して送信

**返答できない質問の場合：**
- 「幹事に確認して折り返します」と返す
- 幹事のダッシュボードに「参加者からの質問」として通知

---

### 機能D：会の振り返りヒアリング（幹事向け）

**トリガー：** イベントの終了時刻から2時間後（自動）

**フロー：**
1. KANJIが幹事にLINEでメッセージを送る
2. 幹事が返答する（自由テキスト）
3. 返答内容をAIが構造化してDBに保存
4. 次回イベント作成時に「前回の振り返り」として参照できる

**ヒアリングの流れ（会話形式）：**

```
KANJI：
お疲れ様でした！今日の飲み会どうでしたか？🍺
よかったら一言教えてください（何でもOKです）

幹事：
めちゃ盛り上がった！お店も最高だった

KANJI：
いいですね！お店の評価を教えてもらえますか？
⭐⭐⭐⭐⭐（タップで選択）

幹事：（星4つをタップ）

KANJI：
ありがとうございます！
次回また使いたいお店ですか？

幹事：
絶対また行く

KANJI：
了解です！「また行きたい店」リストに追加しておきますね✅
他に何かメモしておきたいことはありますか？

幹事：
田中が遅刻してきた（笑）

KANJI：
メモしました😄
今日のまとめをダッシュボードに保存しておきます！
```

**保存される情報：**
```
event_reviews（イベント振り返りテーブル）
  - event_id
  - organizer_id
  - free_text（幹事の自由記述）
  - restaurant_rating（1〜5）
  - want_to_revisit（また行きたいフラグ）
  - structured_memo（AIが構造化したJSON）
  - created_at
```

**蓄積データの活用：**
- 「また行きたい店リスト」として幹事ダッシュボードに表示
- 次回イベント作成時に「前回使ったお店」として候補に表示
- フェーズ2の「おすすめ店舗」ページの口コミデータとして活用

## 3. ユーザー体験の全体フロー（時系列）

```
【イベント前】
  幹事：イベント作成（3分）
    ↓
  KANJI：過去の参加者リストを表示
    ↓
  幹事：誰に送るかチェック → 送信
    ↓
  参加者：LINEに招待メッセージが届く（幹事の名前で）
    ↓
  参加者：URLを踏んで登録 → KANJI公式を友達追加
    ↓
  KANJI：友達追加を検知 → 幹事の連絡先リストに追加

【イベント前日】
  KANJI：参加者全員に「明日の飲み会リマインド」を自動送信

【イベント当日】
  参加者：「集合場所どこ？」とLINEで質問
    ↓
  KANJI：イベント情報を参照して自動返答

【イベント後〜翌日】
  KANJI：幹事に「振り返りヒアリング」を送信
    ↓
  幹事：会話形式で返答
    ↓
  KANJI：構造化してDBに保存

【精算時】
  KANJI：未払い者に自動リマインド
    ↓
  参加者：「支払いました」をタップ
    ↓
  KANJI：幹事に通知
```

## 4. 実装優先順位

| 優先度 | 機能 | 理由 |
|--------|------|------|
| ★★★ | 参加者リストの自動蓄積（機能A） | 他の全機能の前提。これがないと何もできない |
| ★★★ | 友達追加状況の可視化 | 幹事が「誰にLINEで送れるか」を把握するために必須 |
| ★★☆ | 振り返りヒアリング（機能D） | 幹事の継続利用に直結。実装コストも低い |
| ★★☆ | 招待メッセージの自動送信（機能B） | 「誘うところから」の体験。友達蓄積後に実装 |
| ★☆☆ | 参加者からの質問への自動返答（機能C） | 便利だが必須ではない。フェーズ2以降でOK |

---

## 5. 技術スタック

| 要素 | 技術 |
|------|------|
| LINE友達追加検知 | LINE Messaging API Webhook（フォローイベント） |
| メッセージ送信 | LINE Messaging API（Push Message） |
| 招待文・返答文の生成 | LLM（invokeLLM / 既存のBuilt-in API） |
| 振り返りの構造化 | LLM（JSON Schemaで構造化出力） |
| 会話の文脈管理 | DBにセッション状態を保存（state machine） |
