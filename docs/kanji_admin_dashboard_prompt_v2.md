# Claude Code 向け：Admin ダッシュボード（事業 KPI 管理）の実装プロンプト

現在の `AdminDashboard.tsx` は、システム全体の「累計データ（ストック指標）」を表示する設計になっています。これを、事業の成長（グロース）とボトルネックの特定を目的とした「アクティブデータ（フロー指標）」中心のダッシュボードに改修してください。

以下のプロンプトをコピーして Claude Code に渡してください。

---

```text
AI KANJI プロジェクトの `AdminDashboard.tsx` を、事業の成長（グロース）とボトルネックの特定を目的とした「アクティブデータ（フロー指標）」中心のダッシュボードに改修してください。

以下の要件に沿って、`AdminDashboard.tsx` と必要な Supabase のクエリ・スキーマ（必要に応じて）を修正してください。

### 1. 最上部：事業の健康状態（KGI と バイラル指標）
以下の数値を最上部のカード（Grid）で表示し、可能であれば「前月比（MoM）」の増減も表示してください。

- **MAU（月間アクティブ幹事数）**: 今月、1回以上イベントを作成または更新したユーザー数
- **当月の流通総額（GMV）**: 今月作成されたイベントの合計金額
- **総利用者数**: 登録ユーザー（幹事）＋ 招待されたユニーク参加者（ゲスト）の合計
- **バイラル係数（K-factor）**: 既存幹事からの招待URL経由で新規登録した幹事の平均数（※現状トラッキングがない場合は、実装方針の TODO コメントまたはプレースホルダー数値を置いてください）

### 2. 中間部：Activation ファネル（ボトルネックの可視化）
ユーザーが登録してからコアバリューを体験するまでの歩留まり（コンバージョン率）を可視化するセクションを追加してください。棒グラフやパーセンテージ付きのステップUIが望ましいです。

1. **新規登録数（当月）**
2. **オンボーディング完了率**: 登録者のうち、プロフィールの初期設定を終えた割合
3. **初回イベント作成率**: オンボーディング完了者のうち、実際にイベントを1件以上作成した割合

これにより「どこでユーザーが離脱しているか」が一目で分かるようになります。

### 3. 下部：運用・品質指標
サービスが正常に機能しているかを測る指標をリストまたはミニカードで表示してください。

- **精算完了率**: 過去30日間に作成されたイベントのうち、ステータスが「精算完了」になった割合
- **PayPay 番号収集率**: 招待された参加者のうち、PayPay 番号を登録した割合（集金成功の前提条件）

### 実装上の注意点
- 既存の Tailwind CSS クラス（`grid-cols-1 md:grid-cols-3` や `bg-white rounded-lg shadow` など）を踏襲し、UI のトーン＆マナーを統一してください。
- Supabase のクエリが複雑になる場合は、フロントエンドで計算するのではなく、将来的な SQL ビュー（View）化を見据えたクエリ設計にしてください。
- 参加者（Guest）のデータは `participants` テーブルなどを参照し、重複を排除してカウントするロジックを組んでください。
```
